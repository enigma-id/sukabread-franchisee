# Technical Plan: Outlet Catalog Management

**Task ID:** outlet-catalog-management
**Status:** Ready for Implementation
**Based on:** spec.md

## 1. System Architecture
This feature will follow the existing project pattern:
- **Service:** RTK Query for data fetching and mutations.
- **Hook:** A custom hook generated by `createCrudHook` for standardized API interaction.
- **UI:** A table-driven page using the `useTable` hook and standard layout components.

### Architecture Decisions
| Decision | Choice | Rationale |
|----------|--------|-----------|
| State Management | RTK Query | Project standard for API state. |
| Table Logic | `useTable` Hook | Automates pagination and filtering. |
| Update Pattern | Modal Form | Keeps the user in context while performing quick updates. |

## 2. Technology Stack
| Layer | Technology | Rationale |
|-------|------------|-----------|
| Frontend | React | Core framework. |
| Styling | Tailwind CSS | Utility-first CSS for fast UI building. |
| Icons | Lucide React | Existing icon library. |
| API | RTK Query | Efficient data fetching and caching. |

## 3. Component Design
### OutletCatalog Page (`src/pages/settings/OutletCatalog.tsx`)
- **Responsibilities:** Page layout, table initialization, and managing modal visibility.
- **Data Flow:** Uses `useMenu` (catalog hook) and `useTable`.

### CatalogTableConfig (`src/pages/settings/table/catalog.config.tsx`)
- **Responsibilities:** Defines table columns (Name, Min Stock, Max Stock, Status Toggle, Actions).
- **Actions:** Trigger status toggle mutations and open the edit modal.

### StockRangeModal (`src/pages/settings/modal/StockRangeModal.tsx`)
- **Responsibilities:** Form for `min_stock` and `max_stock`.
- **Validation:** Ensures `min_stock <= max_stock`.

## 4. Data Model
```typescript
export interface CatalogOutlet {
  id: number;
  name: string;
  min_stock: number;
  max_stock: number;
  is_active: boolean;
  category_name?: string;
  image_url?: string;
}
```

## 5. API Contracts
| Method | Path | Description |
|--------|------|-------------|
| GET | `/catalog-outlet` | Fetch the list of items. |
| PUT | `/catalog-outlet/:id` | Update stock ranges. |
| PUT | `/catalog-outlet/:id/active` | Set item status to active. |
| PUT | `/catalog-outlet/:id/deactive` | Set item status to inactive. |
